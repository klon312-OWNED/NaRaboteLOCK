============================================================
  БРОНИРОВАНИЕ ДАТ — Инструкция
============================================================

ОПИСАНИЕ
--------
Desktop-приложение (Electron) для бронирования дат сотрудниками.
Читает и записывает данные в таблицу .xlsx (совместима с
Microsoft Excel, Р7 Офис, LibreOffice, МойОфис).

СТРУКТУРА ПРОЕКТА
-----------------
бронирование/
├── package.json          — зависимости и скрипты
├── config.json           — настройки (имя руководителя, лимит)
├── main.js               — Electron главный процесс
├── preload.js            — мост между Node.js и интерфейсом
├── lib/
│   └── schedule.js       — чтение/запись таблицы (.xlsx/.ods)
├── renderer/
│   ├── index.html        — интерфейс
│   ├── styles.css        — стили
│   └── app.js            — логика интерфейса
├── data/
│   └── schedule.xlsx     — файл графика (создаётся автоматически)
├── scripts/
│   ├── setup_windows.bat — настройка прав (Windows)
│   ├── setup_linux.sh    — настройка прав (Linux/Astra Linux)
│   └── create_test_data.js — генерация тестовых данных
└── README.txt            — этот файл

УСТАНОВКА
---------
1. Убедитесь, что установлен Node.js (v18+ рекомендуется):
   https://nodejs.org/

2. Откройте терминал в папке проекта и установите зависимости:

   npm install

3. (Опционально) Создайте тестовые данные:

   npm run create-test

НАСТРОЙКА
---------
Откройте config.json и укажите:

{
  "manager": "имя_пользователя_ОС_руководителя",
  "scheduleFile": "data/schedule.xlsx",
  "maxPerDate": 2
}

- manager — имя пользователя ОС руководителя (Windows: USERNAME)
- scheduleFile — путь к файлу графика (.xlsx или .ods)
- maxPerDate — максимальное количество сотрудников на одну дату

ЗАПУСК
------
   npm start

   или

   npx electron .

РАЗГРАНИЧЕНИЕ ПРАВ В ОС
------------------------

Windows (от имени Администратора):
   scripts\setup_windows.bat

Linux / Astra Linux (от имени root):
   sudo bash scripts/setup_linux.sh

Скрипт ограничит прямой доступ к папке data/ — обычные
сотрудники смогут работать только через приложение.

ПОДДЕРЖКА Р7 ОФИС
------------------
Файл schedule.xlsx совместим с Р7 Офис. Руководитель может
открывать и редактировать его напрямую в Р7 Офис / Excel.

Если нужен формат .ods:
1. Измените в config.json: "scheduleFile": "data/schedule.ods"
2. Приложение автоматически определит формат по расширению.

РОЛИ
----
- Руководитель (совпадает имя ОС с config.manager):
  • Видит все записи
  • Может отменить любую бронь
  • Может открывать файл напрямую

- Сотрудник:
  • Вводит свой табельный номер / ФИО
  • Видит календарь с доступными датами
  • Может записаться на свободную дату (макс. 2 на дату)
  • Видит список своих бронирований

ТЕСТОВЫЕ ДАННЫЕ
---------------
После npm run create-test в data/schedule.xlsx будут:
- Сотрудники: 10001, 10002, 10003
- Даты: апрель 2026 (частично заняты)

Войдите как 10001 / 10002 / 10003 для проверки.
